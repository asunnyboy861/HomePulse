# Pricing Configuration

## Monetization Model: Freemium + IAP (Hybrid: One-Time Purchase + Subscription)

HomePulse uses a **freemium model with hybrid IAP**: a generous free tier for daily use, plus a Pro upgrade available as either a one-time purchase (recommended) or auto-renewable subscription (low-barrier alternative).

**Rationale** (from guide):
- All data processing is local (HomeKit + Home Assistant local communication) — NO third-party API costs
- No recurring server costs — one-time purchase is financially viable
- Free tier is sufficient for 80% of entry-level users — strong user acquisition
- One-time purchase ($6.99) is the recommended option, lower than competitors (HomeDash $9.99, Home+ $20.99)
- Monthly subscription ($0.49) is the lowest in the market — lowers trial barrier

## App Store Connect Setup (Manual — Required)

### Step 1: Create IAP Products
1. Sign in to [App Store Connect](https://appstoreconnect.apple.com)
2. My Apps → HomePulse → In-App Purchases
3. Create 3 products:
   - **Non-Consumable**: `com.zzoutuo.HomePulse.pro.lifetime` at $6.99
   - **Auto-Renewable Subscription (Yearly)**: `com.zzoutuo.HomePulse.pro.yearly` at $1.99/year
   - **Auto-Renewable Subscription (Monthly)**: `com.zzoutuo.HomePulse.pro.monthly` at $0.49/month
4. Add localized display names and descriptions (English US)
5. Add subscription group description for the auto-renewable subscriptions
6. Add review screenshot for each IAP (Paywall screenshot)

### Step 2: Configure Subscription Terms
For auto-renewable subscriptions, include in product description:
- Subscription title, length, price
- Auto-renewal disclosure: "Payment will be charged to your Apple ID account at confirmation of purchase. Subscription automatically renews unless canceled at least 24 hours before the end of the current period. Manage in your App Store account settings."

### Step 3: Submit for Review
- IAP products are reviewed alongside the app binary
- Ensure Paywall UI in app matches IAP product descriptions
- Include Restore Purchases functionality

## IAP Implementation Plan (for PHASE 4+5)

### IAPService.swift (StoreKit 2)
- `Product.products(for: [productIDs])` — fetch product info
- `Product.purchase()` — initiate purchase
- `Transaction.updates` — listen for transaction updates
- `Transaction.currentEntitlements` — verify active entitlements
- `isPro: Bool` — observable property, true if any Pro product active
- `restorePurchases()` — re-fetch transactions

### PaywallView.swift
- Display 3 pricing options (Lifetime highlighted as "Recommended")
- Privacy Policy link (functional, opens Safari)
- Terms of Use link (functional, opens Safari)
- Restore Purchases button
- Manage Subscriptions link (opens App Store settings)
- Auto-renewal disclosure text
- Feature comparison list (Free vs Pro)

### Pro Gating Logic
- `isPro` observed throughout app
- Pro features check `isPro` before showing UI
- If user attempts Pro feature while `isPro == false` → present PaywallView
- All Pro features gracefully degrade to free tier (no broken UI)
